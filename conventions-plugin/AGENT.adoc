== Regles Absolues

=== Git

**INTERDICTION FORMELLE** de commit/push/merge sans permission explicite de l'utilisateur.

Commandes autorisees (lecture seule) : `git status`, `git diff`, `git log`, `git show`
Commandes interdites (sauf ordre) : `git add`, `git commit`, `git push`, `git merge`, `git rebase`

=== Tests

L'agent NE DOIT JAMAIS lancer de tests (`./gradlew test`, etc.) lors de la procedure de fin de session SANS permission explicite de l'utilisateur.

=== Securite
* Jamais de secrets en dur dans le code
* Jamais de credentials dans les fichiers de config
* Jamais de tokens API commites dans Git
* Utiliser des references (`keyRef: ${GOOGLE_API_KEY_1}`)

=== Secrets et tokens — INTERDICTION DE LOG

**PRINCIPE** : Les tokens API, mots de passe, et identifiants credentials NE DOIVENT JAMAIS apparaitre dans la sortie terminal de l'agent. L'utilisateur peut streamer son terminal (Discord, Twitch, etc.). Tout token visible dans le terminal est compromis.

**INTERDICTIONS** :
* **NE JAMAIS** passer un token en argument visible d'une commande
* **NE JAMAIS** afficher le contenu d'un fichier contenant des tokens si le token peut etre visible dans la sortie
* **NE JAMAIS** utiliser `echo`, `cat`, ou tout commande qui affiche des tokens dans le terminal

=== Fichiers de configuration — REGLE ABSOLUE DE SURETE

**PRINCIPE** : Tout fichier de configuration (YAML, JSON, XML, TOML, properties, .env, etc.) peut contenir des donnees sensibles non versionnees.

**Procedure OBLIGATOIRE** avant modification d'un fichier de configuration :
1. `git check-ignore <fichier>` : Verifier si le fichier est versionne
2. `git ls-files -- <fichier>` : Confirmer s'il est tracke
3. Si le fichier **n'est pas tracke** : **CREER UNE SAUVEGARDE** avant toute modification
4. Privilegier **l'edit partiel** plutot que **l'ecriture complete**

=== Methodologie de Travail

* **1 fichier a la fois** : Modifier un seul fichier avant de passer au suivant
* **Validation immediate** : Compiler apres chaque modification
* **Contexte leger** : Maximum 50k tokens EAGER (~3000 lignes)

=== Ordre de Surcharge Configuration (extension API)

Chaque extension de plugin suit la cascade 5 couches :

[source]
----
CLI (-Pproperty=value) > gradle.properties > DSL (extension { ... }) > fichier YAML > defaults
----

* **Couche 1** : CLI (`-Pconventions.xxx=...`) — priorite maximale
* **Couche 2** : `gradle.properties` (`conventions.xxx=...`)
* **Couche 3** : DSL (`conventions { xxx = "..." }`)
* **Couche 4** : Fichier de configuration YAML
* **Couche 5** : Defaults codes en dur dans les data classes

== Session Procedure

=== Debut de Session
1. Lire `PROMPT_REPRISE.adoc` pour la mission en cours
2. Lire `.agents/INDEX.adoc` pour les regles absolues
3. Lire `.agents/CODE_REVIEW_GLOBALE.adoc` pour comprendre le perimetre

=== Fin de Session

**Procedure obligatoire en 6 etapes** :
1. Creer l'archive LAZY : `.agents/sessions/{N}-{type}-{sujet}.adoc`
2. Mettre a jour PROMPT_REPRISE.adoc — Mission N+1 (max 70 lignes)
3. Mettre a jour SESSIONS_HISTORY.adoc
4. Mettre a jour INDEX.adoc — Date, session en cours N+1
5. Mettre a jour TEST_COVERAGE_ANALYSIS.adoc (si tests modifies)
6. Mettre a jour COMPLETED_TASKS_ARCHIVE.adoc

**Aucune etape ne peut etre marquee OK si le fichier n'a pas ete effectivement modifie et verifie.**

| 4 | **Redondance cachee** | INDEX.adoc = source de verite pour roadmap/EPICs
